# Blue Team Labs Online - Log Analysis: Sysmon

## 🎯 Laboratory Objectives
The purpose of this analysis is to investigate a security incident in a Windows environment by analyzing detailed **Sysmon (System Monitor)** logs. The key objectives include:
* Identify persistence or evasion mechanisms utilized by the attacker through environment variable manipulation.
* Trace the deployment of local exploit tools used for privilege escalation.
* Determine network Indicators of Compromise (IoCs), specifically the command and control (C2) channel or *reverse shell*.

---

## 💻 Technical Procedure & Findings

### 1. System Environment Manipulation (Defense Evasion)
While reviewing process creation events (**Sysmon Event ID 1**), it was detected that the attacker executed commands to modify the legitimate system environment variable `COMSPEC` (which by default points to the `cmd.exe` command interpreter).

* **Finding:** The attacker redirected this variable toward a malicious executable located in a temporary directory.
* **Compromised Variable:** `comspec=C:\windows\temp\supply.exe`
* **MITRE ATT&CK Technique:** T1574.007 (Hijack Execution Flow: Path Interception by Runtime Modification).

### 2. Privilege Escalation Binary Analysis
Following the process activity within `C:\windows\temp\`, the presence and execution of the executable binary `juice.exe` was identified.

* **Tool Type:** **Local Privilege Escalation (LPE)** tool, publicly known as **Juicy Potato**.
* **Attack Mechanics:** The tool abused the `SeImpersonatePrivilege` by exploiting the RPC storage service (DCOM) to perform a security token impersonation, elevating local privileges to the highest system level: `NT AUTHORITY\SYSTEM`.



### 3. Reverse Shell Establishment
Once the attacker successfully elevated privileges on the target machine via Juicy Potato, network traffic generated by the tool was tracked to confirm remote access consolidation (**Sysmon Event ID 3**).

* **Attacker IP (C2):** `192.168.1.11`
* **Destination Port (Reverse Shell):** `9898`
* **Outcome:** The malicious binary established a successful outbound connection over port `9898`, granting the attacker an interactive shell with full control over the server.

## 📋 Plan of Action / Investigation Methodology
To reconstruct the attack timeline, the following log analysis framework was applied:
1. **Evasion Filtering & Detection (Event ID 1 - Process Creation):** Monitor the execution of suspicious commands aimed at altering system environment variables (`set`, `comspec`).
2. **Privilege Escalation Analysis:** Investigate anomalous binary executions within temporary directories designed for security token impersonation.
3. **Network Activity Identification (Event ID 3 - Network Connection):** Correlocate identified suspicious processes with unauthorized outbound connections.

---

## 🛡️ Conclusions & Case-Based Recommendations
* **Privilege Mitigation:** Restrict the `SeImpersonatePrivilege` to strictly necessary users and remove it from web service or application accounts that do not explicitly require it.
* **Sysmon Detection Rules:** Implement specific alerts for command-line modifications involving the assignment of values to the `comspec` variable outside of `%SystemRoot%\system32\cmd.exe`.
* **Network Monitoring:** Block unauthorized outbound connections from servers to non-standard ports (such as `9898`) at the perimeter firewall level.
